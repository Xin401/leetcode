# Linked List

## 核心心法

1. **永遠先備份 `next`**：在改變任何指標的 `next` 之前，一定要先用暫存變數（如 `nextTemp`）把它原本指向的節點記下來，否則後面的串列會直接斷裂遺失。
2. **防禦性程式設計 (防 Null)**：使用 `curr->next` 之前，大腦必須反射性檢查 `curr` 是否為 `nullptr`。
3. **沒有頭緒就畫圖**：指標操作是無法在腦海中空想的。卡住時，在紙上畫三個圈圈，跟著程式碼把箭頭擦掉重畫，馬上就能找出 Bug。

## 為什麼需要 Dummy Node？

當你需要**刪除給定串列的第一個節點**時，如果沒有 Dummy Node，你必須寫一段額外的邏輯來更新 `head` 指標。
有了 Dummy Node 擋在最前面，原先的第一個節點就變成了「第二個節點」，所有的刪除、插入邏輯就能完全統一，不需寫任何特殊邊界處理。
