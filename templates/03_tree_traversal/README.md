# Tree Traversal (二元樹走訪)

## DFS 三種走訪的使用時機

1. **前序 (Pre-order, 根左右)**
   - **時機：Top-down (由上往下) 傳遞狀態。**
   - **情境：** 當前節點的邏輯，需要依賴父節點傳下來的資訊時。
   - **例題：** 翻轉二元樹、計算根到葉子的路徑。

2. **中序 (In-order, 左根右)**
   - **時機：針對 Binary Search Tree (BST) 特化。**
   - **情境：** BST 的左子樹皆小於根，右子樹皆大於根。使用中序走訪 BST，必定會得到一個**「由小到大排序」**的陣列。
   - **例題：** 驗證是否為合法 BST、尋找 BST 的第 K 小元素。

3. **後序 (Post-order, 左右根)**
   - **時機：Bottom-up (由下往上) 收集資訊。**
   - **情境：** 當前節點的邏輯，必須等待它的「左子樹」和「右子樹」都算出結果並回傳後，才能決定自己的答案。
   - **例題：** 尋找最近共同祖先、求二元樹最大深度。

## BFS 的使用時機
1. **層級資訊：** 題目明確要求「逐層」處理或回傳資料。
2. **無權重圖的最短路徑：** 尋找從根節點走到某目標的最少步數。
